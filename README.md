# Классификация телефонных записей (vishing)

По аудио **WAV** (в том числе вложенные папки) строится транскрипт на русском (**Kairos ASR**) и выставляется метка:

| `label` | Смысл |
|--------|--------|
| **0** | мошенничество (высокий риск по сценарным правилам) |
| **1** | не мошенничество |

Стек: **Python 3.10+**, **ffmpeg** в `PATH`, модель **kairos-asr** (CPU по умолчанию).

Логика в `scenario_fraud.py`: сценарные признаки + слой **нарушений типичного банковского регламента** (запросы кодов/CVV, «безопасный счёт», AnyDesk и т.д.).

---

## Гайд запуска

### 1. Что должно быть в системе

- **Python 3.10 или новее** (проверка: `python3 --version`).
- **ffmpeg** в `PATH` (на Linux: `sudo apt install ffmpeg` и т.п.).
- При **первом** запуске — **интернет**, чтобы скачались веса Kairos (при лимитах Hugging Face может понадобиться `HF_TOKEN`).

### 2. Установка

```bash
cd vishing
python3 -m pip install -r requirements.txt
```

### 3. Основная команда (любые данные, в т.ч. тестовый датасет)

Положите все `.wav` в одну папку и выполните:

```bash
python3 predict.py /путь/к/папке_с_wav -o results.csv
```

Или явно с порогом по умолчанию:

```bash
python3 predict.py /путь/к/папке_с_wav --threshold 5.5 -o results.csv
```

На выходе **`results.csv`**: разделитель **`;`**, **UTF-8 с BOM**, колонки **`Название файла`** и **`label`**.

### 4. Калибровка порога (размеченные данные)

Порог `--threshold` в `predict.py` задаёт, при каком **fraud_score** запись считается мошенничеством (`label = 0`). Значение по умолчанию (**5.5**) можно **подобрать** на своей разметке.

**Структура каталога:** в корне две папки — **`Fraud`** (положительный класс: мошенничество) и **`NotFraud`** (негатив). Внутри — `.wav` (вложенные папки допускаются).

**Что делает `calibrate.py`:** для каждого файла запускаются те же **Kairos ASR** и **`scenario_fraud`**, что и в `predict.py`; затем по сетке порогов ищется такой **`thr`**, при котором максимален **F1** для класса «мошенничество» (в коде метки: `y=1` в `Fraud`, `y=0` в `NotFraud`). В консоль выводится лучший порог и метрики; дополнительно при **`--reference-thr`** (по умолчанию **5.5**) печатается матрица ошибок **TP / FN / FP / TN** для сравнения с боевым порогом.

Пример:

```bash
python3 calibrate.py /путь/к/корню_с_Fraud_и_NotFraud
```

Полезные опции: **`--threshold-min`**, **`--threshold-max`**, **`--threshold-step`** (диапазон и шаг перебора порога), **`--reference-thr 5.5`**, **`--dump 200`** — вывести фрагменты транскриптов по ошибкам (FN/FP) для анализа.

Подобранный порог передаётся в прогон:

```bash
python3 predict.py /путь/к/wav --threshold <значение_из_калибровки> -o results.csv
```
